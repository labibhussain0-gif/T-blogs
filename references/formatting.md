# 5. FORMATTING RULES

**Structure:**
- Use clear, hierarchical headings (H2, H3, H4).
- Start with a strong hook or executive summary.
- End with a clear conclusion or call to action.

**Readability:**
- Keep paragraphs relatively short (3-4 sentences).
- Use bold text for emphasis on key terms or takeaways.
- Extensive use of bullet points and numbered lists for comparing options or outlining steps.

**Code Snippets:**
- Always provide context before a code block.
- Use syntax highlighting.
- Keep snippets focused; don't paste 100 lines of code if only 10 are relevant.

**Visuals:**
- Diagrams and architecture charts (like Mermaid.js) are highly encouraged.
- Break up large walls of text with meaningful visuals.
